This page tries to explain what we have planned for GNU Radio's future, both long and short term.

In the Near Future

  • Better FFT display widget. At a minimum we have to be able to zoom into signals by setting center frequncy and span, scale the y axis both automatically and manually, handle resize events, and control display update rate (to manage CPU consumption). We may be able to pull this off quickly using the QwtPlot widget and friends.
  • Better oscilloscope display widget. Similar problems to FFT widget. Need to be able to step the time base 1, 2.5, 5, 10 (like a real scope!), triggering, resize, update rate.
  • Make sure that Amateur Radio folks that want to use GNU Radio with sound cards have what they need. At a minimum we need to support complex input from the sound card.
  • Get some digital modulation and demodulation demos working.
  • From an enthusiast:

    I was thinking about this all last night. I'm excited to get the card and tuner. There are about 10 different comms standards I'd like write. An interesting app/demo might be digitizing an entire ham band, like 144-148 MHz, and displaying real-time occupancy, displaying/logging all packet radio data transmissions, DTMF tones, PL tones used, recording all voice transmissions, etc. I'd also like to try Color TV, with sound, and even HDTV.

  • Many modulation techniques: FSK, PSK, QAM, ...

In the Future

  • Scripting interface. Allow C++ signal processing primtives to be glued together on the fly with guile, etc.
  • Standardized interface for plugging in modules that ``find interesting signals''.
  • Look at SCA (Standard Communications Architecture). SCA is a CORBA based standard that is being promoted for military and commercial SDRs.
  • Build something similar to Agilent's 89600 ``Vector Signal Analyzer''
  • Transmitter / Transceiver support
  • Encrypted transceiver for the unlicensed band
  • IS-136, IS-95, GSM
  • A complete cell phone implementation
  • GPS Receiver.
